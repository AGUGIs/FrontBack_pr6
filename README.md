# 🛒 Интернет-магазин товаров - Фронтенд и Бэкенд Разработка

Данный проект представляет собой полноценное веб-приложение интернет-магазина кухонной посуды (кастрюли, ковши, казаны). Проект разработан в рамках практических занятий 4 семестра и включает в себя:
Frontend — клиентская часть на React с использованием SASS
Backend — серверная часть на Node.js + Express
API Documentation — интерактивная документация Swagger (OpenAPI 3.0)
CRUD Operations — полный цикл операций с товарами (Create, Read, Update, Delete)

## 🚀 Запуск проекта

### Требования
- Node.js (версия 16 или выше)
- npm или yarn

### 1. Бэкенд

```bash
# Перейдите в папку backend
cd backend

# Установите зависимости
npm install

# Запустите сервер
node app.js
```
Сервер запустится на http://localhost:3000
📚 Swagger документация API: http://localhost:3000/api-docs

### 2. Фронтенд
```bash
# Откройте новый терминал и перейдите в папку frontend
cd frontend

# Установите зависимости
npm install

# Запустите приложение
npm run dev
```
Приложение запустится на http://localhost:3001

 Выполненные практические работы
### Практика №1 — CSS-препроцессоры (SASS)
## Цель: Изучение SASS/LESS, вложенности, переменных, миксинов
## Реализовано:
1 Переменные (цвета, размеры, отступы)
2 Миксины (@mixin card, @mixin container)
3 Вложенность селекторов
4 Медиа-запросы для адаптивности
5 Модульная структура стилей

### Практика №2 — Сервер на Node.js + Express
## Цель: Создание REST API с использованием Express.js
## Реализовано:
1 Настройка Express-сервера
2 Middleware для парсинга JSON
3 Логирование запросов
4 Обработчики ошибок
5 CRUD-маршруты для товаров

### Практика №3 — JSON и внешние API
## Цель: Работа с JSON, тестирование API в Postman
## Реализовано:
1 Структура JSON-объектов
2 Тестирование API через Postman
3 Валидация данных
4 Обработка ошибок API

### Практика №4 — API + React
## Цель: Связка фронтенда и бэкенда через API
## Реализовано:
1 Axios-клиент для запросов
2 React-компоненты (список, карточка, модальное окно)
3 State-менеджмент (useState, useEffect)
4 CORS-настройка на сервере
5 Обработка загрузки и ошибок

### Практика №5 — Расширенный REST API + Swagger
## Цель: Документирование API с помощью Swagger
## Реализовано:
1 Установка swagger-jsdoc и swagger-ui-express
2 JSDoc-аннотации для всех эндпоинтов
3 Описание схемы Product
4 Интерактивная документация
5 Тестирование запросов через Swagger UI
